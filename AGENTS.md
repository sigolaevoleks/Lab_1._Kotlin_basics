# Workspace Instructions

## Scope
These instructions apply to everything under `/Users/aleksandrsig/IdeaProjects` unless a deeper `AGENTS.md` overrides them.

## Defaults
- Prefer small, focused changes that match the existing project structure and style.
- Read the local code and configuration before changing behavior.
- Do not rename files, move directories, or introduce new tools unless the task clearly requires it.

## Edits
- Keep implementations simple and avoid speculative refactors.
- Update nearby documentation when behavior or setup changes.
- Add comments only when the code is otherwise hard to understand.

## Validation
- Run the narrowest relevant check first when validation is needed.
- If a command is expensive or changes the environment, ask before running it.

## Safety
- Never delete user data or revert unrelated changes without explicit approval.
- Treat uncommitted workspace changes as user-owned unless the current task created them.
